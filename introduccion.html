JAVASCRIPT "LA HISTORIA"


JavaScript fue desarrollado originalmente por Brendan Eich de Netscape con el nombre de Mocha, el cual fue renombrado posteriormente a LiveScript, para finalmente quedar como JavaScript. El cambio de nombre coincidió aproximadamente con el momento en que Netscape agregó compatibilidad con la tecnología Java en su navegador web Netscape Navigator en la versión 2002 en diciembre de 1995. La denominación produjo confusión, dando la impresión de que el lenguaje es una prolongación de Java, y fue considerada por muchos como una estrategia de mercadotecnia de Netscape para obtener prestigio e innovar en el ámbito de los nuevos lenguajes de programación web.




BREVE INTRODUCCION A JavaScript

QUE ES JavaScript?

JavaScript (abreviado comúnmente JS) es un lenguaje de programación interpretado, dialecto del estándar ECMAScript. Se define como orientado a objetos, basado en prototipos, imperativo, débilmente tipado y dinámico.

Se utiliza principalmente del lado del cliente, implementado como parte de un navegador web permitiendo mejoras en la interfaz de usuario y páginas web dinámicas3​ y JavaScript del lado del servidor (Server-side JavaScript o SSJS). Su uso en aplicaciones externas a la web, por ejemplo en documentos PDF, aplicaciones de escritorio (mayoritariamente widgets) es también significativo.

Tradicionalmente se venía utilizando en páginas web HTML para realizar operaciones y únicamente en el marco de la aplicación cliente, sin acceso a funciones del servidor. Actualmente es ampliamente utilizado para enviar y recibir información del servidor junto con ayuda de otras tecnologías como AJAX. JavaScript se interpreta en el agente de usuario al mismo tiempo que las sentencias van descargándose junto con el código HTML.

Desde el lanzamiento en junio de 1997 del estándar ECMAScript 1, han existido las versiones 2, 3 y 5, que es la más usada actualmente (la 4 se abandonó7​). En junio de 2015 se cerró y publicó la versión ECMAScript 6.

LENGUAJE DINÁMICO

JavaScript es un lenguaje de programación dinámico porque permite que las variables puedan tener valores de diferentes tipos de datos durante su ejecución. Esto significa que no es necesario declarar explícitamente el tipo de datos de una variable al principio.

INTERPRETADO

En JavaScript, el intérprete asigna un tipo a una variable en función de su valor en ese momento. Esto se debe a que las comprobaciones de tipo se realizan en tiempo de ejecución, es decir, solo cuando se ejecuta el programa.

ORIENTADO A OBJETOS

JavaScript es un lenguaje orientado a objetos, basado en prototipos, imperativo, débilmente tipado y dinámico. Se utiliza principalmente en navegadores web para mejorar la interfaz de usuario y crear páginas web dinámicas.

FUNCIONAL

JavaScript se considera un lenguaje funcional porque tiene elementos de programación funcional y orientada a objetos, y porque sus funciones pueden tratarse como variables:

  • En JavaScript, todas las funciones son de primera clase, lo que significa que pueden asignarse como valores a variables, devolverse desde otras funciones y pasarse como argumentos a otras funciones.
  • En JavaScript se pueden utilizar objetos inmutables y funciones de primera clase.
  • En JavaScript se pueden realizar cálculos basados en la programación orientada a objetos.

En general, los lenguajes funcionales se basan en el concepto de función, y priorizan el uso de recursividad y funciones de orden superior para resolver problemas.

ASÍNCRONO

JavaScript es un lenguaje de programación asíncrono porque permite que un programa inicie una tarea larga y siga respondiendo a otros eventos mientras se ejecuta.

La asincronía es un pilar fundamental de JavaScript, ya que es un lenguaje de un solo subproceso, lo que significa que solo puede ejecutar una cosa a la vez. Si no fuera asíncrono, no podría realizar operaciones largas, como acceder a la red, sin bloquear el hilo principal.

La programación asíncrona es necesaria porque muchas de las funciones de los navegadores pueden tardar mucho tiempo en ejecutarse. Por ejemplo, acceder a la cámara o micrófono de un usuario, realizar peticiones HTTP o pedir a un usuario que seleccione archivos.

Algunos patrones asincronos de JavaScript son:

  • CALLBACK: Se ejecuta cuando una operación asíncrona termina.
  • PROMESA: Representa el resultado de una operación asíncrona.
  • ASYNC/AWAIT: Permite manejar promesas de una forma más simple.

ECMASCRIPT 6 (ES6) O POSTERIOR

JavaScript es un lenguaje de scripting o interpretado que se basa en el estándar EXMAScript

ECMAScript: Es el estándar que regula cómo debe funcionar y ser interpretado el lenguaje JavaScript

ECMAScript 6 (ES6) es una versión del estándar ECMAScript que incluye novedades como:

  • Nuevas palabras clave para declarar variables, como let y const
  • Nueva sintaxis de funciones mediante funciones de flecha

ECMAScript fue desarrollado por la organización ECma International (European Computer Manufacturers Association) para evitar que cada navegador tuviera una versión diferente del lenguaje JavaScript.

JavaScript es un lenguaje de scripting interpretado que se procesa en diversas plataformas, como los navegadores web y NodeJS.

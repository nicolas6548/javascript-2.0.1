VARIABLES: Var VS Let

Que es una Variable: es un espacio en la memoria para almacenar datos, como números o texto, y se puede referir a ellos con un nombre. Las variables son útiles para guardar y recuperar datos, y para asignar nuevos valores.

Para escribir codigo JavaScript en html tenemos que poner la etiqueta script, y para ver el resultado hay que abrir el inspeccionador del navegador y pasar a la consola del inspeccionar, donde vamos a ver los resultados del script, esa es la forma de hacerlo en html, lo ideal es hacer una hoja solo de JavaScript y hacer un enlace a esa pagina desde el html.

Como se declara una variable?

Utilizamos la palabra reservada VAR junto al nombre de la variable, despues = y en el caso ejemplo vamos a poner un string, por lo tanto ponemos "Hola Mundo" (entre comillas porque es un string), abajo vemos el codigo para que lo copien y lo puedan probar en su navegador

      <script>
      var hola = "Hola Mundo";
      console.log(hola);
    /script>

Si abrimos la consola como mencionamos arriba, vamos a ver que nos sale el famoso "Hola Mundo"

Las variables existen en forma GLOBAL o de BLOQUE

En el código anterior la variable hola es de tipo global, es decir que puede ser utilizada desde cualquier parte del código.

Si quisieramos crear una variable de bloque, utilizamos la palabra reservada LET

Aca vamos a ver un ejemplo con LET

      <script>
      let hello = "Hello world!";
      console.log(hello);
    /script>

Para dejar en claro los las variables con VAR y LET

  • Var es una palabra reservada que se utiliza para declarar variables globales, es decir que se pueden utilizar en cualquier parte del código.

  • Con LET y CONST, se declaran variables locales, es decir que solo se pueden utilizar dentro del bloque en el que se han creado.

ENTONCES, QUE UTILIZAMOS, VAR O LET?

La buena practica dice que NO SE USE MAS VAR NO ES BUENA PRACTICA, porque son variables globales, cuando tu codigo es chico no vas a tener problemas sobre reasignacion de valores, pero si tu codigo crece, por ahi asignas un nombre de variable que ya tenias y le cambias el valor sin darte cuenta, porque VAR es global, y de esa forma vas a tener muchos dolores de cabeza, en cambio si utilizamos LET las variables son en bloque y por lo tanto podemos utilizar el mismo nombre de variable sin correr el riesgo de sobreescribir una variable anterior.

Para que se entienda con un ejemplo porque no es recomendable usar VAR vamos a ver los siguientes ejemplos:

      <script>
      var musica = "ROCK"
      console.log("La variable fuera del bloque nos muestra ", musica);
      {
        var musica = "JAZZ";
        console.log("La variable dentro del bloque muestra ", musica);
      }
      console.log("La variable fuera del bloque nos muestra de nuevo ", musica);
    /script>

EXPLICACION

Creamos la variable musica (usando var) fuera del bloque, despues hacemos un bloque con las llaves { todo lo que esta aca adentro es un bloque en JavaScript }, donde modificamos la variable musica en el bloque, y al salir del bloque hacemos otro console.log para ver que nos toma la variable musica y el resultado es JAZZ, porque la variable con VAR es de hambito global. Ahora vamos a hacer el mismo ejemplo con LET

Ejemplo con LET

      <script>
      let musica2 = "ROCK"
      console.log("La variable fuera del bloque nos muestra ", musica2);
      {
        let musica2 = "JAZZ";
        console.log("La variable dentro del bloque muestra ", musica2);
      }
      console.log("La variable fuera del bloque nos muestra de nuevo ", musica2);
    /script>

Cuando vemos en la consola el resultado de las variables con LET vemos que el let que esta fuera del bloque nos muestra "ROCK", la que esta dentro del bloque nos muestra "JAZZ" y el console.log fuera del bloque nos da como resultado "ROCK", PORQUE LET ES UNA VARIABLE DEL HAMBITO LOCAL, por este motivo dicen que es buena practica usar LET y no VAR



CONSTANTES

Una constantes es un tipo INMUTABLE, NO puede cambiar una vez definida, se usa la palabra const en lugar de var, al igual que let su scope es de bloque, son tipos de sólo lectura y se le debe asignar un valor en el momento de su declaración. Son referencias inmutables, pero sus valores no necesariamente.

     const DIEZ = 10;
            DIEZ = 5;
            console.log(DIEZ); // Imprime Uncaught TypeError: Assignment to constant variable

Aca el error nos dice que "NO SE PUEDE REASIGNAR UNA CONSTANTE" como declaramos la constante en 10, no le podemos decir que despues vale 5, justamente porque es una constante y las constantes no varian.

     const hola = 'hola mundo';
            hola = 'hola mundo'; // Imprime Uncaught TypeError: Assignment to constant variable

En este ejemplo, declaramos la variable "hola" y le asignamos "hola mundo", pero despues al poner hola =, JavaScript lo toma como una reasignacion y sale el error, aunque como se ve la reasignacion es la misma declarada en la constante, pero no se lee la reasignacion, sino que se lee que se quiere hacer una reasignacion nada mas.

     const PI;
            PI = 3.141592653589793; //Imprime Missing initializer in const declaration

Una de las reglas de las constantes dice que "LAS VARIABLES DECLARADAS CON CONST, DEBEN SER INICIALIZADAS EN EL MISMO MOMENTO DE SU DECLARACION".

"NO SE PUEDE DECLARLAR LA CONSTANTE PRIMERO Y ASIGNARLE UN VALOR DESPUES, COMO SI SE PUEDE HACER CON LET O VAR.

¿PORQUE ESTA RESTRICCIÓN?

CONST fue creada para garantizar que una variable nunca cambie después de su creación

Si permitiera asignaciones posteriores, perderia su propósito de ser una "constante".

     const obj = {};
            obj.prop = 'x';
            console.log(obj); //Imprime { prop: 'x' }
            obj.prop = 'y';
            console.log(obj); //Imprime { prop: 'y' }

En este ejemplo se declara un objeto vacio {}

IMPORTANTE: const solo evita la reasignación de la variable obj, pero no protege las propiedades del objeto. Es decir, obj = nuevo_objeto; nos daria error, pero modificar con obj.prop es valido, porque estamos accediendo a la propiedad del objeto en si.

Aunque obj es una constante, sus propiedades pueden ser modificadas, añadidas o eliminadas

En el ejemplo tenemos dos reasignaciones, pero no a la const sino a su propiedad, por eso vemos que hay 2 cambios

     const D = document;
            console.log(D); //Imprime el objeto document
            console.log(D.documentElement); //Imprime el elemento <html>
